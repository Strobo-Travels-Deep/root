# Logbook — 2026-04-13 — Engine v0.9.0 Round 1 fixes

**Context.** Engine audit
([2026-04-13-engine-audit.md](2026-04-13-engine-audit.md)) identified
seven discrepancies vs Hasse 2024. This round implements all six
in-engine fixes (D1–D6) as backward-compatible toggles, all defaulting
to v0.8 behaviour. WP-V Fig 6 and Fig 8 drivers ported to v2 to opt
into the new physics.

Engine version bumped: `0.8.0` → `0.9.0`.

## 1. Patch summary

### `scripts/stroboscopic_sweep.py` v0.9.0 changes

New `DEFAULTS` keys (all default to v0.8 behaviour):

| key                     | default | effect when set                              |
|-------------------------|---------|----------------------------------------------|
| `mw_pi2_phase_deg`      | `None`  | applies MW π/2 about (cos φ x̂ + sin φ ŷ) before AC train (D3) |
| `ac_phase_deg`          | `0.0`   | per-train AC analysis phase: dresses C with e^{iϕ} (D2) |
| `intra_pulse_motion`    | `False` | adds ω_m·a†a to per-pulse H; gap propagator covers (T_sep − δt) (D1) |
| `delta_t_us`            | `None`  | overrides auto-derived δt = π/(2N·Ω_eff) (D6) |

Other v0.9 changes:
- **Header docstring (D5).** Frequency-unit convention clarified:
  `omega_m`, `omega_r` are angular (rad / engine time unit), not cyclic
  MHz. Cyclic-MHz equivalents added to `compute_derived` output
  (`omega_m_cyclic`, `omega_eff_cyclic`, `T_m`, `dt_auto`).
- **Coupling-block convention comment (D4).** Engine uses `(C σ_-, C† σ_+)`;
  Hasse Eq. C1 is `(C† σ_-, C σ_+)`. Comment notes equivalent η-symmetric
  observables and that empirical Fig 8a slope sign matches Hasse.
- **`build_hamiltonian` signature extended.** New keyword args
  `ac_phase_rad`, `omega_m`, `intra_pulse_motion`. Existing positional
  callers unaffected.
- **`run_single` rewires Ufree.** When `intra_pulse_motion=True`, free
  evolution is applied for `T_sep − δt` between pulses (the gap), since
  the per-pulse propagator now consumes δt of real time. When
  `intra_pulse_motion=False`, original v0.8 logic preserved: Ufree only
  built for `t_sep_factor ≠ 1.0`.
- **`_mw_pi2_apply` helper.** Applies a closed-form 2×2 spin rotation to
  the prepared state before the AC train, identity on motional. No
  matrix multiplication; vectorised slice arithmetic.

Backward compatibility: WP-E `run_slices.py` and any caller relying on
`DEFAULTS` produces bit-identical results to v0.8 (verified at
α=3, ϑ₀=42°: σ_z = +0.128438 in both versions, diff zero).

### Fig 8 v2 — α-sweep with intra-pulse motion

`run_fig8_v2.py` enables D1+D3 on the same |α|, tilt grid as v0.8.
Wall: 3 s.

What changed: the spin observable σ_z is now strongly α-dependent
(no longer flat at +0.128). |C| oscillates instead of being flat:
e.g. tilt 0°, |C|(α) = 0.035 → 0.85 (α=2) → 0.03 (α=4) → 0.85 (α=6)
→ 0.06 (α=8). Per-pulse spin rotation is no longer a clean π/2 — it
varies with α because intra-pulse motional rotation modulates the
effective coupling.

What did **not** change to match Hasse: the *shape* of the contrast
decay. Hasse Fig 8b is monotone decreasing 1 → 0; engine v0.9 is
oscillatory. The two are in different regimes:

| Quantity        | Hasse              | Engine v0.9 |
|-----------------|--------------------|-------------|
| N pulses        | 30                 | 22          |
| δt              | 100 ns (~13 % T_m) | ~41 ns (~5 % T_m) |
| ω_m·δt per pulse| 0.82 rad (≈ 47°)   | 0.34 rad (≈ 19°) |

Hasse's larger ω_m·δt smears the AC kick over a larger fraction of a
motional cycle per pulse, producing the Bessel-function-like suppression
that gives smooth monotonic decay. Engine v0.9 sees less smearing per
pulse and instead exposes coherent revival structure (the spin re-aligns
periodically as the motional kick accumulates and unwinds).

Plots: [fig8a_position_v2.png](../plots/fig8a_position_v2.png),
[fig8b_momentum_v2.png](../plots/fig8b_momentum_v2.png).

## 4. Round 2 plan (proposed)

Use D6 to set δt = 100 ns explicitly (matching Hasse), and re-derive N
by requiring N·Ω_eff·δt ≈ π/2:

```
N = π / (2 · Ω_eff · δt)
```

With current Ω = 0.3 (engine angular units, ≈ 0.048 MHz cyclic), η = 0.397,
Ω_eff ≈ 0.273, δt = 0.1 (engine units corresponding to 100 ns at the
"angular = MHz cyclic × 2π" reading): N ≈ 57. That's a lot of pulses.

Alternative: set δt ≈ 0.13·T_m = 0.628 (engine units) and N = 22, giving
total budget N·Ω_eff·δt ≈ 22·0.273·0.628 = 3.77 rad — well past π/2.
Need a careful re-derivation.

Recommendation: implement Round 2 as a *parameter-matching exercise* —
write a small driver that scans (N, δt) at fixed ω_m·δt = 0.82 rad
(Hasse target) and finds the (N, Ω) combination whose accumulated
rotation stays at π/2. Then re-run Fig 8b at that point.

Not gating; logged as the next step.
