- **Root cause** (identified, see §2 below): the engine runs in the

  the inter-pulse propagator `Ufree = None` and the motional state does
  not rotate between pulses. The Hamiltonian during each pulse contains
  only the carrier coupling Ω/2·(C ⊗ σ₋ + h.c.), no motional H_m =

  in Hasse's Fig 3a inset is zero, and the contrast loss mechanism that
  Hasse identifies as the origin of Fig 8b is structurally absent.

## 2. Diagnosis: stroboscopic-frame contrast loss is not in the engine

The engine's per-pulse propagator at `t_sep_factor = 1.0`:

```
H = (Ω/2) [exp(iη(a + a†)) ⊗ σ₋  +  h.c.]      ─ no ω_m a†a term
U = exp(−i H δt)
```

In Hasse's lab frame the same pulse contains the motional Hamiltonian and
the wave packet rotates by ω_m · δt ≈ 0.13 · 2π during each flash. That
rotation, accumulated over the train, produces the position smearing that
suppresses contrast for high |α| where the wave packet's tangential
displacement during δt is comparable to the AC wavelength.

The engine therefore implements a more idealised (and faster) protocol —
the "perfectly synchronous, zero-duration flash" limit — than Hasse's
finite-δt train. This is consistent with the engine's docstring scope
("stroboscopic detuning-scan simulation") but is a *load-bearing
simplification* that previous WPs may not have flagged.

### What this means for WP-E

WP-E reads C(δ₀, |α|, φ_α) from the same engine. The Doppler-detuning
axis δ₀ is fully captured (it is in the Hamiltonian explicitly). The
*amplitude* axis |α| in the engine carries:

- correct η-nonlinear phase shift (Fig 8a confirms);
- **no contrast decay** at δ₀ = 0 (Fig 8b refutes).

WP-E S1 (δ₀, |α|) plots showed contrast variation; that variation must
therefore come entirely from the Doppler axis (Rabi-lineshape filter), not
from the motional smearing channel. This is internally consistent but
sharpens the WP-E narrative: in this engine, position is encoded in
arg C and momentum is encoded in |C|(δ₀)|, with the (δ₀ = 0, |α|) axis
nearly degenerate. The Hasse paper attributes amplitude-axis contrast
loss to position smearing; the engine attributes it to Doppler filtering
alone.

### What this means for the engine

Two mutually exclusive readings:

(a) **The engine is correct for the rotating-frame Hamiltonian it
declares.** Hasse's contrast loss requires lab-frame finite-δt physics
that the engine intentionally factors out. Add a `lab_frame=True` mode
that injects ω_m a†a into the per-pulse H to recover Fig 8b. *Recommended.*

(b) **The engine silently approximates** by setting Ufree = None at
t_sep_factor = 1.0, and the t_sep_factor != 1.0 path (which builds Ufree)
is the right one. Test by re-running Fig 8b with `t_sep_factor = 0.999`
and checking whether contrast decays. If yes, t_sep_factor = 1.0 is a
silent shortcut; if no, the issue is the missing intra-pulse motional
term and (a) is the fix.

Recommendation: a one-line follow-up run with t_sep_factor = 0.999 to
discriminate (a) vs (b). Not gating WP-V; the result is informative
either way.
