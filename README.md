## Simulate page (v0.6)

Visitors can run the full stroboscopic detuning scan in their browser:

- **Parameters:** alpha, eta, omega_m, Omega, N_max, detuning range, pulse count
- **Decoherence:** Spin T1, T2, motional heating (dn/dt) via quantum trajectories
- **Projection noise:** Configurable N_rep with binomial sampling and error bars
- **Convergence diagnostics:** Fock truncation checked post-run with colour-coded banner
- **Downloads:** JSON, CSV, Python/QuTiP script, manifest — all with SHA-256 provenance hash

## Data

- `data/alpha_*/detuning_scan.h5` — Pre-computed HDF5 from adaptive-learner simulations
- `data/manifest.json` — HDF5 dataset manifest
- `data/runs/*.json` — Default simulation runs (generated by simulate.html engine)
- `data/runs/manifest.json` — Simulation run manifest

## Provenance

Every simulation result carries a SHA-256 hash covering: code version, repository URL,
all parameters (including decoherence and noise settings), timestamp, output array
fingerprints, and Fock convergence diagnostics. This hash enables traceability if results
are merged into the open-research-platform database.

## Architecture notes

- `simulate.html` is self-contained (inline CSS/JS) for portability. For a production
  merge into the Harbour site, the CSS should be extracted to `style.css`.
- The numerics viewer (`js/numerics-viewer.js`) handles three data sources: HDF5 (via
  jsfive), JSON runs, and user uploads.
- Default runs in `data/runs/` use the same physics parameters as the HDF5 data
  (alpha = 0, 1, 3, 5) for cross-validation. Note: alpha = 5 with N_max = 40 shows
  1.1% Fock leakage — this is genuine and the convergence banner correctly flags it.
